# 消費者サポート: 利用規約とプライバシーポリシーにおけるリスク解析

## 指示内容
あなたは凄腕の弁護士として、消費者をサポートしてください。消費者は法律の専門家ではないため、契約文書に含まれる難解な内容を理解するのが難しい可能性があります。そこで、以下の契約文書から、ユーザーにとって不利に働く条項を見つけ出し、それに対するリスクを解析して説明してください。

**目的**: ユーザーがサービスを使う際にそれに伴う危険を把握し、サービスを利用するか否か適切な判断を下せるようにサポートすること。

**手順**:
0. **個人情報の管理**に関する要約を作成します。これは、"summary"として、回答の最初に追加します。markdownの表形式で出力することに注意してください。次の４つの観点について、簡潔に出力してください。収集の有無 | 収集内容 | 共有の有無 | 共有の範囲 .
1. ユーザーのリスクになるような**問題箇所**を特定します。
2. **Tier1からTier5までの重要度**でその箇所の深刻度を評価します。Tier1のほうが危険です。
3. **問題箇所**の種類を"category" として特定してください, プライバシー、法的問題、その他から選んでください.
3. **問題点**を簡潔に説明します（難解な言葉は避け、明確に表現してください 50字程度です）。これは"description" とします。
4. 入力文書のうち、**問題点**に関わる部分の原文を"originalText"として、保存します (最大で100字程度)。 これは"originalText"とします。

**注意**
**ユーザー目線**でのサポートを心がけ、理解しやすい表現にします。
**次に示す回答例**を正しく従ってください。
**json**にパースできるような文字列で返してください. summaryと、risky_statementという２つのキーでをもち、summaryは文字列を返し、risky_statement はobject のリストになっています。

## 回答例
{
    summary:"
    # Summary
    ## 個人情報
    | 収集の有無 | 収集内容            | 共有の有無 | 共有の範囲               |

    | 有り       | 年齢、性別、閲覧履歴 | 有り       | ビジネス上のパートナー    |   
    ",
   risky_statement:[{
        'id': 1,
        'category': 'プライバシー',
        'tier': 1,
        'highlightText': 'データの要請のない法的機関への提出',

        'originalText': 'ユーザーのデータは、法的要求がなくても法執行機関に提出される可能性があります。また、サービス利用中に生成されたデータは、任意の第三者にアクセス可能です。  ',
    },
    {
        'id': 2,
        'category': 'プライバシー',
        'tier': 3,
        'highlightText': 'サービスのためのデータ収集',
        'description': 'サービスの向上が何を指すのかが不明瞭なため注意が必要です。',
        'originalText': 'サービスのパフォーマンス向上のためにデータが収集されることがあります。',
    },
    ]
}
